# OpenDP Verified — Blueprint

A [leanblueprint](https://github.com/PatrickMassot/leanblueprint) for the
handwritten Lean development in `rust/opendp_verified/src/**/*.lean`, which
verifies the Aeneas-extracted Rust samplers against
[SampCert](https://github.com/Shoeboxam/SampCert) reference distributions.

The point of this blueprint is to make **what is proved, what is a gap, and what
is still a trusted axiom** visible at a glance — as a dependency graph and as
the status table below — so a contributor can see where to plug in.

It is purely additive: it documents the existing Lean, touches none of it, and
does not enter the `lake` build. (`Generated/` is regenerated by Aeneas; this
directory is never generated.)

## What the pipeline is

```
src/**/*.rs --Charon--> opendp_verified.llbc --Aeneas--> Generated/OpenDP/*.lean
                                                               |
       handwritten src/**/*.lean  -- prove:  samplerDist(extracted fn) = SampCert PMF
```

Aeneas extracts Rust into the **deterministic** `Result` monad; all randomness
enters through one stochastic external, `fill_bytes` (OpenSSL). The development
is layered as: a semantic foundation that gives every `Result` program a
sub-probability distribution (`functor.lean`), a trust boundary of deterministic
external specs plus the one randomness axiom, then one module per sampler.

## Status legend

| Badge | Meaning | Graph colour |
|---|---|---|
| **proved in Lean** | statement + proof complete | green |
| **GAP: statement only** | stated in Lean, proof stubs at a named step | not-ready |
| **axiom / trust boundary** | a declared `axiom` (trusted assumption) | green leaf, labelled |
| **stub: not yet stated** | no Lean statement exists yet | absent (recorded as intent) |

Auditing tip: `#print axioms <theorem>` enumerates the axioms an end-to-end
theorem actually depends on.

### The two gaps worth contributing to

1. **`exp1` loop-tail bridge** — the unit negative-exponential's per-step law and
   PMF values are proved, but the all-iterations distribution is not yet tied to
   `probWhile` of SampCert's loop body. (Not blocking: `exp` is proved without
   it.)
2. **`noisy_top_k` end-to-end privacy** — sampler correctness is proved and
   SampCert's permute-and-flip privacy theorem is imported, but nothing yet
   connects the extracted Rust mechanism to that ε-DP guarantee.

## Building

Requires the blueprint toolchain (separate from `lake`):

```bash
pip install leanblueprint        # pulls in plasTeX
# from this directory:
leanblueprint web                # HTML + dependency graph -> ../web
leanblueprint pdf                # PDF (needs xelatex) -> print
leanblueprint checkdecls         # validate \lean{...} names against the built lib
```

`checkdecls` requires a green `lake build OpenDPVerified` so the declaration
names can be checked against the compiled library. Until the toolchain is set
up, `content.tex` is authored to match the real decl names harvested from the
source, so the prose and table above are accurate even unbuilt.

      web.tex          # web-only macros
      print.tex        # print-only macros
